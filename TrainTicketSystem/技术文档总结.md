## 4 张核心表

| 表名          | 说明                 | 核心字段                                                     |
| ------------- | -------------------- | ------------------------------------------------------------ |
| `train_info`  | 列车车次信息         | train_id（车次编号）, departure（起始站）, arrival（终点站）, departure_time（发车时间）, price（价格）, total_seats（总座位） |
| `seat_status` | 座位区间状态（核心） | **seat_bitmap**（位图，用0和1规定座位出售状态）, default_mask（起始位图全为0）, train_id（车次编号）, carriage_no（车厢编号）, seat_no（座位号） |
| `orders`      | 订单交易             | order_id（订单编号）, buy_mask（用0和1规定该票购买的路段）, **order_status + status**（双写一份给用户，一份给机器，代表车票状态）, price（票价） |
| `users`       | 用户权限             | username（用户名）, password（密码）, role_type（ADMIN/USER）（判断权限） |



## API 接口一览（共 9 个端点）

### 1. `GET /` — 首页

无请求/响应字段，返回 HTML 页面。

------

### 2. `GET /api/trains` — 查询所有车次

**请求参数：** 无

**响应字段（JSON 数组）：**

| 字段名           | 类型   |
| ---------------- | ------ |
| `train_id`       | str    |
| `train_no`       | str    |
| `departure`      | str    |
| `arrival`        | str    |
| `departure_time` | str    |
| `price`          | number |
| `total_seats`    | int    |

------

### 3. `GET /api/seats` — 查询座位

**请求参数（Query）：**

| 字段名     | 类型 | 说明                |
| ---------- | ---- | ------------------- |
| `train_id` | str  | 可选，默认 `"G101"` |

**响应字段（JSON 数组）：**

| 字段名         | 类型 |
| -------------- | ---- |
| `seat_id`      | int  |
| `carriage_no`  | int  |
| `seat_no`      | str  |
| `seat_bitmap`  | int  |
| `default_mask` | int  |

------

### 4. `POST /api/add_train` — 添加车次（需 ADMIN）

**请求体（JSON）：**

| 字段名           | 类型   | 必填 | 别名                |
| ---------------- | ------ | ---- | ------------------- |
| `train_id`       | str    | ✅    | —                   |
| `train_no`       | str    | ❌    | —                   |
| `departure`      | str    | ✅    | `departure_station` |
| `arrival`        | str    | ✅    | `arrival_station`   |
| `departure_time` | str    | ❌    | —                   |
| `price`          | number | ❌    | —                   |
| `total_seats`    | int    | ❌    | —                   |

**响应字段：**

- 成功: `{"ok": true, "train_id": "..."}`
- 403: `{"error": "forbidden"}`
- 400: `{"error": "train_id, departure and arrival required"}`

------

### 5. `POST /api/buy` — 购票（核心事务）

**请求体（JSON）：**

| 字段名        | 类型    | 必填 | 说明                                        |
| ------------- | ------- | ---- | ------------------------------------------- |
| `train_id`    | str     | ✅    | 车次 ID                                     |
| `carriage_no` | int     | ❌    | 默认 1                                      |
| `seat_no`     | str     | ✅    | 如 `"01A"`                                  |
| `buy_mask`    | int/str | ✅    | 位掩码，支持 int / 十进制字符串 / `"0b..."` |
| `user_id`     | str     | ❌    | session 获取或默认 `"WEBUSER"`              |

**响应字段：**

- 成功: `{"success": true, "order_id": "ORDxxxxxxxxxx"}`
- 失败: `{"success": false}` (HTTP 400)

------

### 6. `POST /api/refund` — 退票（需 ADMIN）

**请求体（JSON）：**

| 字段名     | 类型 | 必填 |
| ---------- | ---- | ---- |
| `order_id` | str  | ✅    |

**响应字段：**

- 成功: `{"ok": true}`
- 400: `{"error": "order_id required"}`
- 403: `{"error": "forbidden"}`

------

### 7. `POST /api/login` — 登录

**请求体（JSON）：**

| 字段名     | 类型 |
| ---------- | ---- |
| `username` | str  |
| `password` | str  |

**响应字段：**

- 成功: `{"success": true, "role": "ADMIN"|"USER", "username": "..."}`
- 失败: `{"success": false, "message": "账号或密码错误"}`

------

### 8. `POST /api/logout` — 登出

无请求/响应字段（仅返回 `{"ok": true}`）。

------

### 9. `GET /api/financial` — 财务统计（需 ADMIN）

**请求参数：** 无

**响应字段（JSON 数组，来自视图 `v_daily_financial_report`）：**

| 字段名            | 类型   | 说明     |
| ----------------- | ------ | -------- |
| `sale_date`       | date   | 销售日期 |
| `train_id`        | str    | 车次     |
| `total_orders`    | int    | 总订单数 |
| `paid_orders`     | int    | 已支付数 |
| `refunded_orders` | int    | 已退款数 |
| `total_sales`     | number | 总销售额 |
| `refund_amount`   | number | 退款总额 |
| `net_revenue`     | number | 净收入   |

------

## 二、数据库表字段

| ---------------- | ------------- | -------- |

### `seat_status`（座位状态）

| 字段名         | 类型        | 约束               |
| -------------- | ----------- | ------------------ |

| `seat_no`      | VARCHAR(10) | NOT NULL           |
| `seat_bitmap`  | INT         | DEFAULT 0          |
| `default_mask` | INT         | DEFAULT 0          |

唯一约束: `(train_id, carriage_no, seat_no)`

| 字段名         | 类型          | 约束                           |

| `buy_mask`     | INT           | DEFAULT 0                      |
| `price`        | DECIMAL(10,2) | DEFAULT 0                      |
| `order_status` | VARCHAR(20)   | DEFAULT 'PAID'                 |
| `status`       | INT           | DEFAULT 1 (1=PAID, 2=REFUNDED) |
| `create_time`  | TIMESTAMP     | DEFAULT CURRENT_TIMESTAMP      |

### `users`（用户）

| 字段名        | 类型         | 约束                      |
| ------------- | ------------ | ------------------------- |
| `user_id`     | VARCHAR(50)  | PK                        |
| `username`    | VARCHAR(50)  | NOT NULL, UNIQUE          |
| `password`    | VARCHAR(100) | NOT NULL                  |
| `role_type`   | VARCHAR(20)  | DEFAULT 'USER'            |
| `create_time` | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP |

## 五、已知缺陷

1. **`index.html` 车站名显示 bug**：前端用 `t.departure_station` / `t.arrival_station` 读取，但 API 返回的是 `departure` / `arrival`，导致下拉选项显示 `(undefined -> undefined)`。
2. **`buyTicket()` 函数是空桩**：前端购票按钮只弹出 `alert()`，未实际发送请求到 `/api/buy`。
3. **密码明文存储**：`users.password` 以明文存储，仅适用于演示环境。
