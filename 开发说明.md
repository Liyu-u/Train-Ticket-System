# 火车站售票管理系统 (Train Ticket System) - 技术说明文档

## 1. 系统概述
本项目是一个模拟现代大型铁路客运系统的售票引擎。系统重点攻克了传统关系型数据库在**高并发秒杀**与**复杂区间计算**场景下的性能瓶颈，通过底层存储结构的优化与事务的精细化隔离，实现了高度一致、防超卖的售票业务闭环。

## 2. 系统架构
### 2.1 技术栈
* **后端核心:** Python 3.x
* **持久层 (RDBMS):** MySQL 5.7+ (InnoDB 引擎)
* **中间件驱动:** PyMySQL

### 2.2 部署架构
系统采用云原生标准交付，整体服务将部署至**阿里云 (Alibaba Cloud) 服务器**上。
* **应用服务器:** 运行 Python 业务服务，处理前端交互、参数校验与路由分发。
* **数据服务器:** 在阿里云服务器上通过 Docker 容器化运行 MySQL 实例，确保环境隔离与高可用性。配合云平台的安全组策略，严格控制 3306 端口的访问权限。

## 3. 核心数据库设计
### 3.1 基于位图 (Bitmap) 的区间售票模型
放弃传统的“一票一座”非黑即白记录方案，采用二进制位图完整映射列车的行驶区间。
* **逻辑模型:** 若列车途经 $N$ 个站点，则有 $N-1$ 个行驶区间。
* **物理实现:** 在 `Seat_Status` 表中引入 `seat_bitmap` (INT) 字段。通过位运算（`&` 与 `|`）实现区间的动态占用与验证。例如，当新购票请求到达时，系统只需验证 `(当前状态 & 请求区间) == 0`，即可判断是否发生座位冲突，将复杂的路径计算降维为极速的底层 CPU 运算。

### 3.2 核心表结构与分区优化 (DDL 摘录)
